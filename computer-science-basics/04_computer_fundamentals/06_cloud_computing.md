## 🧠 Core Concepts Learned

## Cloud Computin Fundamentals 
- Cloud computing replaces traditional physical servers with on-demand online infrastructure  
- It allows businesses and users to access computing resources such as storage, servers, databases, and applications over the internet  
- Cloud services are designed to be flexible, scalable, cost-efficient, and easier to manage than maintaining physical hardware  

### Key Benefits 
- **Scalability**
    - Resources can be increased or reduced depending on the needs of the application  

- **On-demand self-service**
    - Users can create, configure, or remove services when needed 

- **Pay-as-You-Go**
    - Customers are charged based on the resources they use rather than buying expensive physical hardware 

- **Security**
    - Cloud providers protect the infrastructure with strong security measures

- **High Availability**
   - Services are designed to stay accessible even if one part of the system fails  

- **Global Access**
   - Applications and services can be accessed from almost anywhere with an internet connection  

## Types of Cloud
- Cloud computing offers different deployment and usage models to match different needs for control, security, and scalability

### Deployment Types (The Environment)
- **Public Cloud**
    - Services are provided over the internet 
    - Commonly used by startups, websites, and global applications
    - Affordable, easy to scale, and does not require physical infrastructure management  

- **Private Cloud**
    - Cloud infrastructure is dedicated to a single organisation  
    - Often used by banks, healthcare providers, and government organisations  
    - Offers greater control, customisation, and support for sensitive data  

- **Hybrid Cloud**
    - Combines public and private cloud environments  
    - Allows organisations to keep sensitive systems private while still using public cloud resources when needed  
    - Common in businesses that need both security and scalability 

### Main Cloud Service Models (Level of Control)
- Cloud services also differ in how much control the customer keeps over the system and how much is managed by the provider

💡 Cloud computing follows a **shared responsibility model**, where the provider secures the infrastructure, while the user is responsible for configuring and securing their own services correctly  

- **IaaS (Infrastructure as a Service)**
    - Analogy: Renting an empty apartment  
    - Provides basic computing resources such as virtual machines, storage, and networking  
    - The user manages the operating system, applications, and data, while the provider manages the physical infrastructure 

- **PaaS (Platform as a Service)** 
    - Analogy: Renting a semi-furnished apartment
    - Cloud provider manages the infrastructure and the operating system
    - The user focuses on developing, deploying, and running applications without worrying about hardware

- **SaaS (Software as a Service)**
    - Analogy: Staying in a hotel
    - The provider manages everything, and the user simply access the software through a browser or app
    - Examples include Zoom, Gmail, and Microsoft 365

### Real-World Usage
- Most modern applications and services run on cloud infrastructure (streaming platforms, banking apps, social media)  
- Businesses use cloud computing to deploy applications quickly and scale globally  
- Cybersecurity teams monitor cloud environments to detect threats, analyse logs, and respond to incidents  

### Major Cloud Vendors
- **Amazon Web Services (AWS)**
    - The industry leader, offering a wide range of services and global infrastructure  
- **Microsoft Azure**
    - Widely used in enterprise and strong in hybrid cloud solutions
- **Google Cloud Platform (GCP)**
    - Known for powerful data analytics, AI, machine learning services
- **Alibaba Cloud**
    - Is offering competitive cloud services globally
- **IBM Cloud**
    - Focuses on hybrid cloud and AI-driven solution for businesses
- **Oracle Cloud**
    - Focuses on enterprise software, databases, and business infrastructure

## 🔐 Security Relevance
- Cloud environments are common targets for attackers because they host valuable systems and data  
- Misconfigured cloud resources can expose sensitive information to the public  
- Access control is critical, as weak permissions can allow unauthorised access to services and data  
- Logging and monitoring are important in cloud environments to detect suspicious behaviour and support incident response  
- Understanding cloud infrastructure is essential in cybersecurity, as many modern businesses rely on cloud-based systems  

## 📌 Lessons Learned
⚠️ Cloud computing does not remove the need for security, it changes how responsibility is shared between the provider and the user  
⚠️ Cloud services make systems more cost-efficient, flexible, and scalable, but misconfiguration can create serious security risks
