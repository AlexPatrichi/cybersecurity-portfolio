## 🧠 Core Concepts Learned 

## How Websites Work
- When you visit a website, your browser sends a request to a web server requesting a specific resource (such as a webpage or file)
- The web server responds with data that your browser uses to render the page  

### Client–Server Model
- **Client (Browser):** Sends requests to access resources  
- **Server (Web Server):** Processes requests and returns responses  

**Client → Request → Server**  
**Server → Response → Client**  

💡 Web browsers use this model automatically every time you access a website  
💡 Developer tools allow you to view raw HTTP communication between client and server  

⚠️ A web server is a system that hosts websites, processes requests, and delivers content to users over the internet  

## Website Structure
### **Frontend (Client-side):**
  - What users see and interact with in the browser  
### **Backend (Server-side):**
  - Processes client requests, executes application logic, and interacts with databases to generate responses

## Technologies Used in Websites
### HTML (HyperText Markup Language)
- This language defines the structure of a webpage  
- Uses elements (tags) to organise content  

### CSS (Cascading Style Sheets)
- Language used to style and design webpages
- Controls how HTML elements look (colors, fonts, layout)

### JavaScript 
- Adds interactivity to webpages  
- Enables dynamic content (forms, buttons, animations)  
- One of the most widely used programming languages in the world

## 🔐 Security Relevance
- Understanding how websites operate is essential for identifying vulnerabilities  
- Many attacks target the interaction between client and server      

## 📌 Lessons Learned  
⚠️ Websites rely on structured communication between client and server  
⚠️ If this communication is not properly secured, it can be exploited by attackers  
⚠️ Understanding this interaction is the first step in identifying web vulnerabilities
