## 🧠 Core Concepts Learned

## 🛡️ Security Principles
- Security principles are guidelines used to design and maintain secure systems  
- They help reduce vulnerabilities and limit the impact of attacks  

💡 These principles support all areas of cybersecurity, from system design to incident response  

###  Least Privilege
- Users are given only the access they need to perform their tasks  

💡 Reduces the risk of misuse and limits damage if an account is compromised  

### Defense in Depth
- Security is implemented in multiple layers rather than relying on a single control  

**Examples:**
- Firewall + Antivirus + Access Control  
- Authentication + Encryption + Monitoring  

💡 If one layer fails, others still provide protection  

### Zero Trust (Basic Concept)
- No user or system is trusted by default, even inside a network  
- Every access request must be verified  

💡 "Never trust, always verify"  

### Separation of Duties
- Critical tasks are divided among multiple users or roles  

**Example:**
- One user approves transactions, another executes them  

💡 Prevents abuse of power and reduces risk of insider threats  

### Principle of Minimising Attack Surface
- Reduce the number of entry points an attacker can exploit  

**Examples:**
- Disable unused services  
- Remove unnecessary software  
- Limit open ports

## 🔐 Security Relevance
- Security principles guide the design of secure systems and networks  
- Applying these principles reduces the likelihood and impact of attacks  
- Helps organisations build resilient and secure infrastructures  
- Used in risk management, system design, and security policies  

## 📌 Lessons Learned
⚠️ Security should be implemented in layers, not as a single control  
⚠️ Limiting access reduces the potential damage of attacks  
⚠️ Trust should always be verified, not assumed  
⚠️ Strong security design is based on principles, not just tools
