## 🧠 Core Concepts Learned
## Access Control
- Access control is the process of restricting access to systems, resources, and data  
- It ensures that only authorised users can perform specific actions  

💡 Access control works together with **authentication and authorization** to secure systems  

### Access Control Process
1. **Authentication** → Verify identity (who you are)  
2. **Authorization** → Determine permissions (what you can do)  
3. **Access Control** → Enforce those permissions  

### Access Control Models
#### Role-Based Access Control (RBAC)
- Access is based on the user’s role within an organisation  
- Example:
  - User → basic access  
  - Admin → full system access  

#### Least Privilege
- Users are given only the access they need to perform their tasks  

💡 Reduces the risk of misuse or accidental damage  

#### Discretionary Access Control (DAC)
- The owner of a resource decides who can access it  

#### Mandatory Access Control (MAC)
- Access is controlled by a central authority based on security levels  
- Often used in high-security environments

## 🔐 Security Relevance
- Prevents unauthorised access to sensitive data and systems  
- Limits the impact of compromised accounts  
- Reduces risk of privilege escalation attacks  
- Helps enforce organisational security policies  

💡 Restricting admin privileges reduces the damage an attacker can cause if an account is compromised  

## 📌 Lessons Learned
⚠️ Access control ensures that users can only access what they are permitted to   
⚠️ Authentication and authorization are required before access control is enforced    
⚠️ Least privilege is a key principle for reducing security risks    
⚠️ Proper access control limits the impact of security breaches
